## v0.14.3 — 修复连线「没连上去」（2026-06-25）

### 🔧 连线锚点对齐到节点真实大小
- **现象**：画布上把节点连到镜头节点，线看着没接到节点上、飘在节点下方一点点的空中。
- **根因**：连线的落点一直按节点的「名义大小」算，但镜头节点（视频/图片）实际是**按画面比例渲染**的——名义高 340，按 16:9 实际只渲到 236 高。于是线的终点锚在 340 的一半处，比真实端口低了约 52px → 落在节点下方空中，看着「没连上」。
- **修复**：连线的起点 / 终点 / 拖拽预览线，全部改用节点**真实渲染大小**（与节点外壳同一套算法），名义大小不再参与。这类「线落在节点框外」的偏差从任何地方都不会再出现。

### 🔧 参考槽容量修复
- **现象**：给图生图节点点「+」加参考图，图加不进去；连线连进来也「连了等于没连」。
- **根因**：参考槽「还剩几格」算错了——负责显示/发送的逻辑把连线（含还没出图、只占位的连线）一起算进上限，但「要不要显示 +／能不能加」的逻辑只数了有图的和手动上传的，**漏算了那些占着位的「待生成连线」**。于是槽其实满了却以为还有空：`+` 照样显示、图也"存进去了"，可没有空位放它 → 加了不显示、发不出去；新连线落进满槽被丢弃。
- **修复**：「槽满没满」改为只由一个算法说了算（含占位的待生成连线）。槽真满时「+」正确隐藏、硬加给明确提示「参考已满（最多 N，含连线）」，不再默默丢。槽没满时一切照旧。

### 🔧 中转出图 404 修复
- **现象**：部分中转（如 code.newcli.com）接的图片模型一出图就报 `404`，地址长成 `.../codex/v1/v1/images/generations`——多了一段 `/v1`。
- **根因**：项目里有两套拼 URL 的逻辑悄悄走偏了。一套（接入测试用）会在「Base 末尾已带版本段、接口路径又以 `/v1/` 开头」时去掉多余的 `/v1`；但真正发图走的那套漏了这步去重，于是 Base `.../codex/v1` 拼上 `/v1/images/...` 就成了双 `/v1` → 中转找不到这个地址 → 404。
- **修复**：去重逻辑下沉到唯一的拼接器（生成和接入测试共用一套），删掉走偏的平行实现。这类「双版本段」错误从任何接口都不会再出现。

### 🎙️ 中转也能接配音了
- 之前火山原生配音要你**自己的火山 APP_ID / ACCESS_KEY**，没有就用不了。现在用你**已有的中转 key**（如 code.newcli.com）就能加 `seed-tts-2.0` 配音模型——走标准 OpenAI 兼容接口（`/v1/audio/speech`），出火山音色 mp3，即接即用。

## v0.13.1 — 修复打包版「一键接入 Claude Code / Codex」连不上（2026-06-24）

### 🔌 一键接入：握手「连接已关闭」修掉
- **现象**：在 Nomi 点「一键接入 AI 编程助手」后，Claude Code / Codex 连不上（报 `Connection closed` / MCP 握手失败）。
- **根因**：写进客户端的启动命令指向一个**没被打进安装包**的脚本，且打包版不保证有 `node` 可用——进程一启动就退出，客户端在握手第一步就断开。
- **治本**：改成让 Nomi 用**自身程序**直接当 MCP 服务启动（不依赖外部 node、入口永远在包内）。Nomi 开着时实时同步、生成弹确认卡；没开时也能驱动。
- ⚠️ **升级后请重点一次「一键接入」**：旧配置不会被自动改写，重点一次即覆盖成正确入口（Claude Code 与 Codex 同理）。
