## v0.16.2 — 图生图「不按原图」根治（2026-07-06）

### 🖼️ 中转站图生图/参考图，老接入自动就位
- **不用再「删了重加」**：v0.16.0 起新接入的中转图片模型才有图生图，老接入的一直缺——现在启动时自动补齐（catalog v5 迁移）：参考图槽、图生图通道、比例 + 清晰度（1K/2K/4K）全部就位。云雾等 OpenAI 兼容中转的「参考图用不了 / 只能出 1K」到此为止。

### 📌 参考图永不过期
- **参考图一律优先用本地持久文件**，服务商临时直链（多数 1–3 天过期）只做兜底；发送时链接过期会**自动用本地文件重新上传换新链**。老项目重开后「参考图加载失败 / 服务商拉不到图」这一整类问题失效。

### 🛡️ 不再「静默出一张无关的图」
- 图生图/图生视频**缺参考图或该模型没有图生图通道时，在扣费之前直接报错**说明怎么办——此前会被静默当成纯文生图发出去：生成成功、扣费成功、结果和原图毫无关系（「图生图没按原图改」的主要来源）。
- 图生图模式没给参考图时生成按钮直接禁用并提示（对齐视频节点既有做法）；失效的参考图缩略图从含糊的「加载失败」改为「图已失效」+ 恢复提示。

### 接入本地 ComfyUI（2026-07-04）

#### 🖥️ 用本机 ComfyUI 出图，无需 key、不花额度
- **模型设置里新增「本地 ComfyUI」接入卡**：在本机把 ComfyUI 跑起来（默认 `127.0.0.1:8188`），点「启用本地 ComfyUI」即可——无需任何 key，用你自己的 GPU 和模型出图，数据不出本地。默认关，不用的人零打扰。
- **启用时当场探测**：点启用会 `GET /system_stats` 检测本机 ComfyUI 是否在跑，连上就显示 Python 版本 / 显卡 / 显存（连不上也不阻断，可先启用再起服务，生成前不通会直接报错、不白跑）。接入地址可改（跑在别的端口/主机也行）。
- **内置一个「文生图」工作流**：启用后在生成画布直接选「本地 · 文生图」，填提示词 + 尺寸/步数/CFG/采样器/种子/checkpoint 名即可出图（用的是 ComfyUI 原生 `POST /prompt` → 轮询 `/history` → 取 `/view`，不经任何云端）。
- 底层是**通用「命名响应变换」钩子**：让任何形状古怪的上游响应（如 ComfyUI 动态 prompt_id 键 + 拼 /view URL）都能归一后走现有生成链路，未来接别的本地/自建后端可复用。

### 中转站模型「勾选启用」+ 供应商管理（2026-07-04）

#### 🔌 一堆拉取模型，勾选留哪几个
- **中转站模型可就地勾选启用/停用**：接一个中转站往往拉来几十上百个模型，此前只能在 chip 墙里逐个点 × 删（不可逆、删了要重拉）。现在模型设置里展开中转卡就是一份**可搜索、可勾选的清单**——搜模型名、按类分组、全选/全不选、实时「已启用 X/N」。

## v0.14.3 — 修复连线「没连上去」（2026-06-25）

### 🔧 连线锚点对齐到节点真实大小
- **现象**：画布上把节点连到镜头节点，线看着没接到节点上、飘在节点下方一点点的空中。
- **根因**：连线的落点一直按节点的「名义大小」算，但镜头节点（视频/图片）实际是**按画面比例渲染**的——名义高 340，按 16:9 实际只渲到 236 高。于是线的终点锚在 340 的一半处，比真实端口低了约 52px → 落在节点下方空中，看着「没连上」。
- **修复**：连线的起点 / 终点 / 拖拽预览线，全部改用节点**真实渲染大小**（与节点外壳同一套算法），名义大小不再参与。这类「线落在节点框外」的偏差从任何地方都不会再出现。

### 🔧 参考槽容量修复
- **现象**：给图生图节点点「+」加参考图，图加不进去；连线连进来也「连了等于没连」。
- **根因**：参考槽「还剩几格」算错了——负责显示/发送的逻辑把连线（含还没出图、只占位的连线）一起算进上限，但「要不要显示 +／能不能加」的逻辑只数了有图的和手动上传的，**漏算了那些占着位的「待生成连线」**。于是槽其实满了却以为还有空：`+` 照样显示、图也"存进去了"，可没有空位放它 → 加了不显示、发不出去；新连线落进满槽被丢弃。
- **修复**：「槽满没满」改为只由一个算法说了算（含占位的待生成连线）。槽真满时「+」正确隐藏、硬加给明确提示「参考已满（最多 N，含连线）」，不再默默丢。槽没满时一切照旧。

### 🔧 中转出图 404 修复
- **现象**：部分中转（如 code.newcli.com）接的图片模型一出图就报 `404`，地址长成 `.../codex/v1/v1/images/generations`——多了一段 `/v1`。
- **根因**：项目里有两套拼 URL 的逻辑悄悄走偏了。一套（接入测试用）会在「Base 末尾已带版本段、接口路径又以 `/v1/` 开头」时去掉多余的 `/v1`；但真正发图走的那套漏了这步去重，于是 Base `.../codex/v1` 拼上 `/v1/images/...` 就成了双 `/v1` → 中转找不到这个地址 → 404。
- **修复**：去重逻辑下沉到唯一的拼接器（生成和接入测试共用一套），删掉走偏的平行实现。这类「双版本段」错误从任何接口都不会再出现。

### 🎙️ 中转也能接配音了
- 之前火山原生配音要你**自己的火山 APP_ID / ACCESS_KEY**，没有就用不了。现在用你**已有的中转 key**（如 code.newcli.com）就能加 `seed-tts-2.0` 配音模型——走标准 OpenAI 兼容接口（`/v1/audio/speech`），出火山音色 mp3，即接即用。
- 原生火山配音（带情感控制）**保留并存**，两条路按各自能力出现，不互相替代。
