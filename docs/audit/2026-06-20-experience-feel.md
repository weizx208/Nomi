# E5 多维体验质感审计 — 结果 2026-06-20

> 方法：`docs/plan/2026-06-20-experience-feel-audit.md`（启发式 + 感知性能 + 反应卡）。
> 采集：常驻驱动 + 注入探针（延迟/帧率/密度/对比度，见 `tests/ux/ui-driver.mjs`），真机走 J1/J3/J5（J2/J4 复用同画布/节点/助手基建，体验信号等价）。
> 决策：建探针✅ / 只测 UI 反馈质量（不跑真实生成）/ 全 J1–J5。
> 环境：120Hz ProMotion 显示器（故满帧基线=120fps，非 60）。

## 总评

**Nomi 的体验底子很硬**：UI 响应快到「跟手」（多数交互 8–37ms 首反馈，远低于 Doherty 400ms），自研画布平移/缩放满帧无 jank，色彩对比度全面超 WCAG AAA。**主要短板集中在两类**：① 少数主线程计算型交互有可见卡顿（时间轴 scrub、整理画布）；② 没有任何加载占位（skeleton），目前靠「够快」兜底，但重项目/慢网下会暴露；③ 多窗口累积、画布边缠绕等结构性摩擦。无 P0。

## D1 感知性能（等待时间）— 优秀

实测「click → 首视觉反馈 / 内容沉降」（ms）。Doherty 阈值：首反馈 <400ms，长操作 >1s 须有进度。

| 动作 | 首反馈 | 沉降 | 加载占位 | 评 |
|---|---|---|---|---|
| 新建空白项目 | 69ms | 116ms | 无 | ✅ 远超阈值 |
| 打开重项目(45节点+图) | 20ms | **444ms** | 无 skeleton | ⚠️ 沉降略超 400ms，但首帧快、可接受 |
| 切「预览」tab | 18ms | 32ms | — | ✅ |
| 切「生成」tab | 20ms | 37ms | — | ✅ |
| 开模型设置面板 | 22ms | 35ms | — | ✅ |
| 开创作助手面板 | — | — | 119.8fps 滑入 | ✅ 动画顺 |
| AI 发送→用户气泡 | **8ms** | — | 流式+可停止 | ✅ 即时 |
| AI 拆镜头(长等待~20s) | ··· 思考点→流式方案卡 | — | 流式=持续进度 | ✅ 长等待有反馈+可取消 |
| 导出 MP4(18s 片) | 进度条 0→100% | ~6s | 有进度% | ✅ 落有效 1080p |

**结论**：感知性能是强项。最长的两个等待（AI 生成、导出）都有持续反馈（流式 / 进度条）且可取消——教科书级。
**唯一隐患**：全程**无 skeleton 骨架屏**。当前靠「加载够快(<500ms)」兜底，但 ① 重项目 444ms 已贴阈值 ② 慢网/大项目会暴露空白闪烁。2026 业界已把 spinner 视为反模式、skeleton 为标配——这是「现在不痛但该补」的项。

## D2 体验丝滑（响应/运动）— 整体优秀，2 处卡顿

实测交互期间 FPS / long task / 最大帧隙（满帧=120）。

| 交互 | FPS | long task | 最大帧隙 | 评 |
|---|---|---|---|---|
| 空闲基线 | 120 | 0 | 10ms | ✅ |
| 画布平移(拖) | 119.6 | 0 | 25ms | ✅ 跟手 |
| 画布缩放 | 120 | 0 | 19ms | ✅ 丝滑 |
| 创作助手面板滑入 | 119.8 | 0 | 10ms | ✅ |
| **时间轴 scrub** | 112.9 | 0 | **91ms** | ⚠️ scrub 中偶发一顿(~5帧) |
| **整理画布(自动布局)** | 114 | **1 (53ms)** | **122ms** | ⚠️ 触发瞬间明显一顿 |

**结论**：核心画布交互（平移/缩放/面板动画）**满帧无 jank，自研画布性能优秀**。
**2 个弱点**（都是主线程同步计算阻塞渲染）：
- **整理画布**：布局算法在主线程跑出 53ms long task → 122ms 帧隙，触发瞬间可见一顿。
- **时间轴 scrub**：拖动播放头偶发 91ms 帧隙（取帧/重绘开销）。
→ 修复方向：布局/取帧计算移出主线程（Web Worker / 分帧 / requestIdleCallback），或加节流。

## D4 认知负荷（舒不舒服）— 合理

实测每屏可见可点元素数 / 文本量。

| 屏 | 可点元素 | 文本字数 | 评 |
|---|---|---|---|
| 项目库 | 52 | 1123 | 卡片网格，识别为主，OK |
| 生成画布 | 79 | 1809 | 最高可点数，但空间化画布（非线性列表），靠位置识别，可接受 |
| 预览 | 24 | 1469 | 清爽 |
| 创作区 | 31 | 3388 | 文本最多——写作面，预期内 |

**结论**：密度合理。画布 79 个可点元素看着多，但都是节点上的操作锚点（连线/删除/参数），空间分布、按需聚焦，不构成「一眼一堆按钮」的压迫。符合「识别优于回忆」。

## S2 专家评审（两个独立 agent，高度收敛）

### ⭐ 最强信号：两个独立视角都把「画布」判为头号问题
- **设计师**：「静态界面克制得体，**一进画布就混乱掉档**」；画布蓝线毛线球是「廉价感/AI 味」重灾区，「远比任何排版细节严重」。
- **真实用户**：「**画布是整个产品的体验悬崖**」「从漂亮的分镜卡片一脚踩进一团蓝色连线，落差太大，最可能直接关掉 app」。
- 两者独立得出同一结论 → 这是 E5 审计的**第一优先级体验债**，可信度极高。

### D3 视觉美感（设计师评审摘录）
| 屏 | 严重度 | 问题 |
|---|---|---|
| 画布 canvas2 / j5-nomi | **重** | 蓝色直线毛线球交叉缠绕，视觉噪音压倒内容；节点全是无差别灰卡分不清哪个镜头；minimap 形态与主画布不一致 |
| 项目库 / 素材库 / 素材面板 | **中** | 缩略图三形态混排（真实图 / 黑底标题卡 / 纯灰空格子）+ 铺满灰占位格 → 强烈「假数据/未完成」廉价感 |
| 分镜方案卡 j1-split3 | 中 | 全套最挤一屏；左列表+右气泡+蓝框卡三种卡片语言并置；长描述无行动价值（违 R2） |
| 预览 j5-exportview | 中 | 预览放「定妆拼图」非当前帧；clip 缩略图用黑底合成标题卡（创作工坊/生成画布）像软件截图混进素材 |
| 模型下拉 | 中 | 纯文字下拉、无供应商图标/分组，原生 select 廉价感；弹层背景遮罩不足，背后斜纹+蓝线透出显脏 |
| 创作编辑器 | 中 | 内容少时 80% 纯白空场，呼吸过头变空洞 |
| 首页/顶栏 | 轻 | 主 CTA 与次 CTA 权重几乎相等没拉开；「上手3/4」紫色徽章是纯灰黑配色里唯一突兀色 |

**设计师整体判断**：克制的骨架 + 失控的画布 = 当前「半高级」。修掉①画布布线降噪 ②缩略图/空态统一，能从「平庸偏混乱」直接拉到「克制高级」。

### D5 摩擦/情感（真实用户反应卡）
- **情感画像 5 词**：专业、高效（AI 真能干活）、有压力、复杂、**忽冷忽热**（头尾友好、中间吓人）。
- **最劝退点**：写完文案 → 不知道下一步点哪（无「拆镜头」明显入口，引导断层）→ 确认后跌进节点画布毛线球。
- **最惊喜点**：AI 把大白话文案自动拆成 9 个带画面的镜头 + 助手用人话解释 →「它替我想好了」的魔法感，是用户愿意继续的唯一理由。
- **关键诉求**：给「只想出片、不想懂节点」的用户一条**绕过画布的傻瓜直达路径**（分镜卡 → 直接预览导出），画布作为高级选项藏起来，而非逼新手正面硬刚蓝线。
- 其它友好/可信屏：预览导出（最安心）、首页（最友好）。
- 模型选择：认得 Sora/Veo 有面子，但**无「新手用这个/哪个快哪个便宜」指引** → 选择困难。

## Top-N 不爽清单（按修复 ROI 排序）

1. **🔴 画布「毛线球」降噪**（D3+D5，两 agent 共识 #1）—— 边曲线/正交布线 + 默认降透明度 + 仅 hover 高亮当前节点的边 + 节点显示镜号/标题/状态。**最高优先级体验债**。（已知问题，见 [[canvas-tidy-layout]] [[canvas-experience-and-edge-semantics]]）
2. **🔴 给「不想懂节点」用户傻瓜直达路径**（D5）—— 分镜卡确认后，除「落画布」外提供「直接生成并预览」路径，画布转为高级可选。产品级取舍，需拍板。
3. **🟠 缩略图/空态系统统一**（D3）—— 项目库/素材库/素材面板统一缩略图比例 + 空态用单一占位插画替代铺满灰格子，消「假数据」感。
4. **🟠 写完文案的下一步引导**（D5）—— 编辑器选中文字/写完后给明确「让 AI 拆镜头」入口，补引导断层。
5. **🟠 模型下拉加新手指引**（D5）—— 推荐项/「新手用这个」/速度·价格提示 + 供应商图标分组。
6. **🟠 预览/时间轴用真实帧**（D3）—— clip 缩略图与预览区放当前帧，删合成标题卡。
7. **🟡 D2 两处卡顿**（时间轴 scrub 91ms / 整理画布 53ms long task）—— 计算移出主线程或节流。
8. **🟡 skeleton 骨架屏**（D1 隐患）—— 重项目/慢网补占位，当前靠「够快」兜底。
9. **🟡 多窗口累积 / 节点压工具栏 / 图片工具栏拦截 / 主CTA权重 / 紫徽章**（轻量打磨）。

> 注：1、2、3 是「把半高级拉到高级」的关键三招；其中 1 是纯视觉债（不改产品逻辑），2 是产品取舍（需用户拍板），3 是设计系统统一。
