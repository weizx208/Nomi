# 剧本 → 分镜方案文档 → 确认 → 落画布（一致性主链路）

> 日期：2026-06-13　状态：方案，待拍板。承接 [记忆架构](2026-06-13-memory-architecture.md) 的收敛
> 结论——「角色/场景/道具一致性」不是记忆问题，是「先把跨镜头要一致的东西生成成参考图、再挂到
> 对应镜头」的编排问题。本文把这条端到端流程定下来，**重点定那份「分镜方案文档」长什么样、
> 字段怎么定、怎么改、怎么读回**（对话拍板：这是整条链最该先想清楚的一环）。

---

## 0. 通俗讲清这条流程（对话已逐句拍板）

1. **读剧本**：你在创作区写好剧本 → 画布这边能读到它。
2. **AI 出方案，先当一份文档放创作区给你看（先不碰画布）**：先理出「跨镜头要一致的东西」（角色/场景/道具），再拆分镜（每镜多长、挂哪些参考、提示词）。
3. **你在创作区审 / 改**：不满意直接改，AI 读你改后的版本。
4. **确认后才落画布**：先建「跨镜头的东西」节点并**先生成**（拿到真参考图）→ 再建分镜节点 → 连参考关系。视频默认 **Seedance 2.0**，时长不超模型上限。
5. **先出首镜 → 你看脸确认 → 批量**。

**两条铁律**（对话定）：① 规划免费可改、执行才花钱（审好再落画布）；② 跨镜头的参考图**先生成**，再出镜头。

## 1. 核心：分镜方案文档（IR）—— 本方案的重点

这是整条链的中枢：AI 产出它、用户改它、AI 读回它落地。

**🔒 拍板（决策 A=B 结构化字段视图）**：不做自由 markdown。方案做成**结构化字段视图**——每个锚、每个镜头是一块卡，字段是**离散输入控件**（时长=选择器、参考=锚的多选、提示词=文本框、carrier=「生成参考图/仅提示词」开关），加/删/重排有明确按钮。**字段直接绑定 `StoryboardPlan` 对象**——用户改字段 = 改对象，**没有「文字→结构」的解析这一步，也就没得错**（A 的自由文字改完重解析极脆，用户判定「一定会出错」，弃）。

### 1.1 数据模型

```ts
type StoryboardPlan = {
  title: string
  anchors: PlanAnchor[]   // 跨镜头要一致的东西（先做）
  shots: PlanShot[]       // 分镜
}

type PlanAnchor = {
  id: string                              // 稳定 id（落画布时 = clientId）
  kind: 'character' | 'scene' | 'prop' | 'style'
  name: string                            // 「林夏」「天台」「红书包」「全片风格」
  description: string                     // 标准描述（落 prompt + 定妆/定景模板用）
  carrier: 'visual' | 'text'              // 视觉锚=要生成参考图；文本锚=只写进 prompt
  // style 锚默认 carrier=text + scope=all（每镜常驻）；character/scene/prop 默认 visual + selective
  scope?: 'all' | 'selective'
}

type PlanShot = {
  index: number
  durationSec: number                     // ≤ 该镜所选模型的最大时长（落地时钳住）
  anchorIds: string[]                     // 这镜用到哪些锚（→ 连参考边 / 拼文本）
  prompt: string                          // 可直接生成的提示词（运镜+动作演进，不复述锚的静态描述）
}
```

### 1.2 文档里有哪些内容/字段（B 渲染成结构化卡，下面用 markdown 只是为了让你看清「有哪些字段」，真实是字段控件不是自由文本）

```markdown
# 分镜方案 · 雨夜追凶  ⟨草案 · 可改⟩

## 跨镜头要一致的（先生成这些参考图，再出镜头）
- 【角色】林夏 · 生成参考图
  高中生，齐肩黑发，红色校服；冷峻警戒的气质。
- 【场景】天台 · 生成参考图
  夜晚，水泥护栏，远处城市霓虹；湿冷反光。
- 【道具】红书包 · 生成参考图
  深红双肩书包，侧面有星星挂饰。
- 【风格】全片 · 仅提示词
  冷色调、胶片颗粒感、低饱和。

## 分镜
### 镜 1 · 5 秒
- 参考：林夏、天台、风格
- 提示词：夜雨天台，林夏倚护栏远望城市霓虹，镜头缓缓推进，雨丝在霓虹里发亮。

### 镜 2 · 5 秒
- 参考：林夏、天台、红书包、风格
- 提示词：林夏转身背起红书包向楼梯口走去，跟拍，脚步带起地面积水反光。
```

**字段说明**：
- 锚行格式固定 `- 【类型】名字 · 生成参考图|仅提示词`，下一行缩进是标准描述。`生成参考图`=视觉锚、`仅提示词`=文本锚。
- 镜头块 `### 镜 N · X 秒` + `- 参考：逗号分隔的锚名` + `- 提示词：…`。
- **参考用「名字」引用锚**（人话），读回时按名字匹配到 anchor.id。

### 1.3 怎么改（B：动字段，不动自由文本）

每锚一块卡、每镜一块卡：
- 锚卡：名字（文本）、标准描述（文本框）、carrier 开关（生成参考图 / 仅提示词）、kind（角色/场景/道具/风格）。
- 镜头卡：时长（选择器，选项 = 当前模型支持的时长，**天然不超上限**）、参考（**从锚列表多选**，不手打名字）、提示词（文本框）。
- 加镜/删镜/重排镜、加锚/删锚 = 明确按钮。

### 1.4 怎么读回（B：没有「读回」——字段即数据）

B 下**根本没有「文字→结构」的解析步骤**：字段控件直接绑定 `StoryboardPlan` 对象，用户改字段就是改对象，落画布时直接拿这个对象。**A 的翻车点（重解析）在 B 里不存在。**

剩下的只是**校验**（不是解析）：
- 参考是从锚列表多选的 → 不可能写出对不上的名字（从源头杜绝 §1.2 那种「阿明」笔误）。
- 删了一个锚、但有镜头还引用它 → 该镜头的参考项标红提示，不是去猜。
- 时长是选择器选的 → 不可能超模型上限。

## 5. 风险（对话里我已点明，留档）

## 7. 切片排期（建议）

1. **S1 剧本勾连**：创作区「拆成分镜」入口 → planner 读当前剧本文档。
2. **S2 方案数据模型 + planner 产出**：定 `StoryboardPlan`（§1.1）；planner 输出这个**结构化对象**（不是自由文本）——含锚识别（复用 SKILL 角色别名归并/复用卡）+ 每镜时长/参考/提示词。
3. **S3 结构化字段编辑器（B 的工程主体）**：在创作区把 `StoryboardPlan` 渲染成可改的字段卡（§1.3）——锚卡 / 镜头卡 / 参考多选 / 时长选择器 / 加删重排。字段绑数据，**改即改对象，零解析**。**先过 R8 样张**（这是新的可见 UI）。
4. **S4 落画布**：拿（改后的）`StoryboardPlan` 对象 → 复用轨迹模式落地（建锚→先生成→镜头→连边）+ 默认 Seedance 2.0 + 时长钳位 + 道具锚。
5. **S5 看首镜确认**：首镜先出→看脸→批量（沿用记忆架构样张）。
6. **S6 真机验**：连参考图的镜头真生成，验一致性到底成不成。

每片独立 commit + 五门 + R13 走查；UI 项（S3/S5）先过 R8 样张。

## 8. 回滚 / 验收门

- **回滚**：方案文档是叠加的新入口；不改现有「直接拆镜到画布」老路（两条入口暂并存，老路作回退，稳定后再收）。
- **验收门**：五门全过；S2/S3 补「产出→改→读回→落地」一致性的可复跑断言（同一份文档落地结果稳定）；S5 真机看脸；UI 过 R8。
