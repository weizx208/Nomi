> 2026-06-24。用户要求：参考 fork `jyycool/Nomi` 的暗黑模式，适配我们分叉后的变动，**足够完整**。
> 用户拍板（AskUserQuestion）：① 调色板用 fork 暖灰 oklch + 我补的轨道/强调色；② 默认行为=跟随系统、之后记住选择；③ 3D 导演台 + devlab：UI 翻、3D 渲染色不动。

## 背景与根据（为什么这么做）

- 暗色当年由 fork `f0561348 feat(theme): add dark mode toggle` 引入，合进 PR18，被我们 `84d22601`「砍暗色」整段删除、钉死 light（用户当时说「暗色先不要搞了」）。本次是**反转该决定**，需同步改文档（否则自相矛盾）。
- 整个 App **token-only**：`check:tokens` 硬编码 hex 基线 = 0。**翻 `--nomi-*` token 即翻全局**，组件零改动。
- **运行时 token 真源 = `tailwind.config.ts` 的 `workbenchBasePlugin` addBase**（编译进 `public/tailwind.generated.css`，`index.html` 仅 `<link>` 此文件）。`src/theme/nomi-tokens.css` / `src/styles/globals.css` / `vendor-overrides.css` / `animations.css` **未被任何 import = 死参考文件**；`check-dangling-tokens` 仍把 `nomi-tokens.css` 当定义源扫描。`src/workbench/workbench.css` 是 live（`WorkbenchShell.tsx` import）。
  - ⟹ 暗色 token 块**必须进 `tailwind.config.ts` addBase 才生效**；同步进 `nomi-tokens.css` 保参考一致 + 喂 dangling 门岗。
