# v0.10.0 — 剧本到分镜的主链路 + Agent 大升级

发布日期：2026-06-14

## 一句话

**从「写剧本」到「画布出片」打通一条主链路：AI 就地拆镜头成可编辑的分镜方案、确认后落画布；Agent 全面升级——记得住、连得对、改得动、排得上时间轴。** 自 v0.9.6 起 155 次提交。

## 新功能

### 分镜方案主链路（剧本 → 方案文档 → 确认 → 落画布）
- **就地拆镜头**：创作区选中剧本即可让 AI 提出「分镜方案」（`propose_storyboard_plan`），不弹独立区。
- **方案字段编辑器**：方案以结构化字段呈现（镜头/角色/场景/提示词），可直接改，零解析。
- **一键落画布**：方案确认后转成画布节点 + 参考边，注入默认视频模型/模式，并把镜头时长钳到模型上限。
- **排片到时间轴**：`arrange_storyboard_to_timeline` 按剧本镜序把镜头排上时间轴。

### Agent 升级
- **重启不失忆**：对话工作缓存落盘 + 冷启动回灌；项目记忆卡（EventLog 物化视图）注入上下文，「AI 记得 N 条」。
- **角色身份一致**：别名归并 + 复用画布已有角色/场景卡，避免重复造卡（借鉴 ViMax）。
- **结构化 prompt 骨架**：schema 作唯一真相源。
- **提议=事务**：批量提议原子落地 + 补偿回滚；执行 vs 批准逐字段对账，偏差以「人话 + 原因 +『让 AI 修一下』」卡呈现；节点锁（AI 硬禁 / 用户软门）；整笔撤销。
- **参考边按能力校验**：连边按模型 archetype 参考槽校验，封死轨迹盲连；首/尾帧接力分流，杜绝「封面冒充首帧」。

### 文本与体验
- 文本生成（续写/重写）收口 AI SDK，逐 token 流式渲染；加首字 + 整体超时，封死「永久转圈不报错」。
- 节点生成中统一品牌转圈遮罩；pending 状态统一（spinner / skeleton）。
- 起始页与 onboarding：动作卡片主入口、空库引导态、「30 秒体验 ↔ 模型接入」衔接、工作台三步引导。
- 破坏性操作确认原语（替换 11 处原生 confirm/alert/prompt）。

## 修复（节选）
- **新节点落点不再压旧节点**：画布新建节点用真实包围盒碰撞避让 + 螺旋找空位（替换旧的「整数点等值假避让」）。
- **节点参数面板生成箭头锁死右下角**：恒定最宽卡宽，换模型不漂移。
- **创作/画布助手发送钮锁右下角**。
- **分镜按镜序排布**：钉死「数组序 = 镜序」，防镜头乱序落画布。
- **文本节点标签中文化**（`Text` → `文本`）。
- 批准≡执行：批准时即解析模型/参数/边，根治「执行与批准有出入」整类问题。
- 修 keyframe→video 首帧边被静默丢弃（对账误报根因）。
- 库首启大标题字号、分镜编辑器长描述截断遮挡等多项视觉修复。
